编程语言与计算机四大件其实是密切相关的；

在学完任何一门语言的基本特性之后(变量、逻辑控制语句、面向对象思想)，接下来都需要进入这四个阶段：

  • 熟悉该语言常用的数据结构

    比如C++的vector、set、Map

    Java的List、Set、Map

    Python的tuple、字典…

    这些都是一致的

    LeetCode刷题很重要！！！

  • 熟悉该语言的多线程、多协程开发

    比如C++、Java、Python的多线程开发;

    在开发过程中，都会遇到操作系统中锁、线程并发的思想。

    要求高一点，就是高并发了。

  • 熟悉计算机网络

    互联网的核心就是通信设备之间互联，靠的就是互联网。

    C++、Java、Python都有类似Socket编程的东西，有的语言直接给封装好了。

    现在流行的框架也都是基于Socket延伸的

  • 熟悉数据库

    JDBC 这个在各个语言中都是有的。

    再要求高一点，就是数据库优化之类的了。

在熟悉了这些东西之后

就是要求从源码的层面来了解了。

提到看源码，主要问的就是实现细节，以及用到的设计模式了

Hexo 初步

Hexo安装

环境需求

  • Node.js

    使用 node -v npm -v 验证是否安装成功；

  • 安装 cnpm

    npm 是国外的，安装 Hexo 比较慢，所以换成 cnpm：

    npm i -g cnpm --registry=https://registry.npm.taobao.org

安装 Hexo

  • 使用 cnpm 安装 Hexo

    cnpm i -g hexo-cli；

    验证：hexo -v

Hexo 配置

  1. 新建一个空文件用于存放博客相关文件

    本机下新建E:\Project\HexoBlog 文件夹

    ps: 但凡博客出错无法解决，只要删除重来即可

  2. hexo init

    生成 Hexo 博客相关文件；

    可以发现新建的空文件多了一大堆东西；

    本地验证: hexo s 验证是否能本地运行

  3. 添加个人博客文件

    在 E:\Project\HexoBlog\source\_posts 文件夹下添加个人的 markdown 文件；

  4. hexo g

    刷新一下博客，将上一步放进来的文件刷新到博客中去。

    ps：每次对博客修改，都应当 hexo c、hexo g 使修改生效

    验证：hexo s ,本地验证是否可行

Hexo 部署

本地创建成功，部署到服务器上，方便随时随地访问

  1. 登录 Github ，新建仓库

    注意： 仓库名必须为 github用户名.github.io

  2. 安装插件

    将博客部署到 Github的插件：cnpm i --save hexo-deployer-git

  3. 修改 配置文件 _config.yml
